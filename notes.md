# ရက် ၃၀ စောင့်စရာမလိုဘဲ Paywall (ပိုက်ဆံတောင်းတဲ့ မျက်နှာပြင်) တက်လာမလာ စမ်းသပ်ဖို့အတွက် အလွယ်ကူဆုံး နည်းလမ်း (၂) ခု ရှိပါတယ်။

## နည်းလမ်း (၁) - Firebase Console တွင် လက်ဖြင့် ပြင်ခြင်း (အလွယ်ဆုံး)
1. ၁။ Firebase Console > Firestore Database ကို သွားပါ။
2. ၂။ users collection ကို နှိပ်ပါ။
3. ၃။ ခင်ဗျား စမ်းသပ်နေတဲ့ Email (Document ID) ကို ရွေးလိုက်ပါ။
4. ၄။ ညာဘက်မှာ expiry ဆိုတဲ့ Field ကို တွေ့ပါလိမ့်မယ်။ (ဥပမာ - 2026-01-05T... စသဖြင့် ရှိနေပါမယ်)။
5. ၅။ အဲဒီ ရက်စွဲက နှစ် (Year) ကို 2024 (သို့မဟုတ်) 2023 သို့ ပြောင်းပြီး Update နှိပ်လိုက်ပါ။
6. ၆။ ပြီးရင် သင့် App ကို Refresh လုပ်လိုက်ပါ။

### ရလဒ်:
ရက်စွဲက ကုန်ဆုံးသွားပြီဖြစ်လို့ App က ဝင်ခွင့်မပေးတော့ဘဲ "Upgrade to Premium" ဆိုတဲ့ Payment Box ကြီး တက်လာပါလိမ့်မယ်။

## နည်းလမ်း (၂) - Browser Console ဖြင့် ပြင်ခြင်း (Developer နည်း)
Firebase Console ဖွင့်ရတာ ပျင်းရင် Browser ကနေ လှမ်းပြင်လို့ ရပါတယ်။
1. ၁။ သင့် App ကို ဖွင့်ထားပါ။
2. ၂။ F12 နှိပ်ပြီး Console ကို ဖွင့်ပါ။
3. ၃။ အောက်ပါ ကုဒ်ကို Copy ကူးပြီး Console မှာ Paste လုပ်၊ ပြီးရင် Enter ခေါက်လိုက်ပါ။

JavaScript
// ရက်စွဲကို မနေ့ကရက်စွဲ (Expired) သို့ ပြောင်းခြင်း
```const yesterday = new Date();
yesterday.setDate(yesterday.getDate() - 1);

db.collection('users').doc(auth.currentUser.email).update({
    expiry: yesterday.toISOString()
}).then(() => {
    console.log("Date updated to expired! Refresh now.");
    location.reload();
});```

### ရလဒ်:
Page က အလိုလို Refresh ဖြစ်သွားပြီး Paywall တက်လာပါလိမ့်မယ်။

## ငွေချေစနစ်ကို စမ်းသပ်ခြင်း (Payment Test)
- Paywall ပေါ်လာပြီဆိုရင် အခုလို ဆက်စမ်းနိုင်ပါတယ် -
- Plan ရွေးပါ: ဥပမာ - Monthly ($2.99) ကို နှိပ်ပါ။

PayPal ခလုတ်ပေါ်လာမယ်: (ခင်ဗျားက Client ID အမှန်ထည့်ထားရင် အစစ်ပေါ်ပါမယ်။ စမ်းသပ်ချင်ရင် Sandbox ID သုံးရပါတယ်၊ ဒါပေမယ့် ကိုယ့်ကတ်နဲ့ကိုယ် Test လုပ်ရင် $2.99 လောက်က ပြန် Refund လုပ်လို့ရပါတယ်)။

- Payment အောင်မြင်ရင်: "Payment Successful" ဆိုပြီး Alert တက်လာပါမယ်။
- App ပြန်ပွင့်လာမယ်: Database မှာ သွားပြန်ကြည့်ရင် expiry ရက်စွဲက နောက်တစ်လ (2026) ကို အလိုလို တိုးသွားတာ တွေ့ရပါလိမ့်မယ်။

ဒီနည်းနဲ့ စနစ်တစ်ခုလုံး အလုပ်လုပ်/မလုပ် စစ်ဆေးနိုင်ပါတယ်ခင်ဗျာ။

## ကုဒ်ကို ကာကွယ်ခြင်း (Code Obfuscation)
အပေါ်မှာ Right-click ပိတ်ထားပေမယ့် ကျွမ်းကျင်သူတွေက ကုဒ်ကို မြင်နိုင်ပါသေးတယ်။ ဒါကြောင့် Javascript Obfuscator သုံးပြီး ကုဒ်တွေကို ဖတ်မရအောင် ရှုပ်ပစ်ရပါမယ်။

### လုပ်ဆောင်နည်း:
- သင့်ရဲ့ app.js ကုဒ်အကုန်လုံးကို Copy ကူးပါ။
- Google မှာ "Javascript Obfuscator Online" လို့ ရှာပါ (ဥပမာ - obfuscator.io)
- သင့်ကုဒ်ကို Paste လုပ်ပြီး Obfuscate နှိပ်ပါ။
- ရလာတဲ့ (နားမလည်နိုင်လောက်အောင် ရှုပ်ထွေးနေတဲ့) ကုဒ်တွေကို ယူပြီး app.js ထဲမှာ ပြန်ထည့်ပြီး Save ပါ။

### သတိပြုရန်: Obfuscate မလုပ်ခင် မူရင်း app.js ကို သီးသန့် Backup လုပ်ထားပါ။ (နောက်မှ ပြင်ချင်ရင် မူရင်းဖိုင်မရှိရင် ပြင်မရတော့ပါ)။

### အနှစ်ချုပ် (Admin ဘယ်လိုလုပ်ရမလဲ?)
- User အသစ်ဝင်လာရင် 30 Days Free အလိုလိုရပါမယ်။
- ရက်ပြည့်သွားရင် သူတို့ App သုံးမရတော့ဘဲ Pricing Table တက်လာပါမယ်။
- သူတို့က ခင်ဗျားဆီ ငွေလွှဲပြီးတာနဲ့ ခင်ဗျားက Firebase Console > Firestore > users collection ကိုသွားပါ။
- အဲဒီ User ရဲ့ Email ကိုရှာပြီး expiry field ကို သူဝယ်တဲ့နှစ်အထိ (ဥပမာ 2026) ပြောင်းပေးလိုက်ရုံပါပဲ။

## ရှင်းလင်းချက် (Summary)
1. Whitelist Collection: "တံခါးစောင့်" ပါ။ (ဝင်ခွင့် ရှိ/မရှိ စစ်သည်)။ ဖျက်လိုက်ရင် လုံးဝ ဝင်မရတော့ပါ။
2. Users Collection: "လက်မှတ်စစ်" ပါ။ (သက်တမ်း ရှိ/မရှိ စစ်သည်)။
သက်တမ်းကျန်ရင် -> App ပေးသုံးမယ်။
သက်တမ်းကုန်ရင် -> Paywall ပြမယ် (PayPal နဲ့ ငွေချေခိုင်းမယ်)။

ဒါကြောင့် စမ်းသပ်ချင်ရင် Whitelist ကို မဖျက်ဘဲ၊ users ထဲက expiry ရက်စွဲကိုသာ ပြောင်းပြီး စမ်းသပ်ပါ။
